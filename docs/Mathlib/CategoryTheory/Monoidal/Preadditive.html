    Preadditive monoidal categories #

    A monoidal category is MonoidalPreadditive if it is preadditive and tensor product of morphisms is linear in both factors.

    source
    class CategoryTheory.MonoidalPreadditive (C : Type u_1) [inst : CategoryTheory.Category C] [inst : CategoryTheory.Preadditive C] [inst : CategoryTheory.MonoidalCategory C] :
    Prop

    • tensoring on the right with a zero morphism gives zero

      tensor_zero : autoParam (∀ {W X Y Z : C} (f : W ⟶ X), CategoryTheory.MonoidalCategory.tensorHom f 0 = 0) _auto✝

    • tensoring on the left with a zero morphism gives zero

      zero_tensor : autoParam (∀ {W X Y Z : C} (f : Y ⟶ Z), CategoryTheory.MonoidalCategory.tensorHom 0 f = 0) _auto✝

    • left tensoring with a morphism is compatible with addition

      tensor_add : autoParam (∀ {W X Y Z : C} (f : W ⟶ X) (g h : Y ⟶ Z), CategoryTheory.MonoidalCategory.tensorHom f (g + h) = CategoryTheory.MonoidalCategory.tensorHom f g + CategoryTheory.MonoidalCategory.tensorHom f h) _auto✝

    • right tensoring with a morphism is compatible with addition

      add_tensor : autoParam (∀ {W X Y Z : C} (f g : W ⟶ X) (h : Y ⟶ Z), CategoryTheory.MonoidalCategory.tensorHom (f + g) h = CategoryTheory.MonoidalCategory.tensorHom f h + CategoryTheory.MonoidalCategory.tensorHom g h) _auto✝

    A category is MonoidalPreadditive if tensoring is additive in both factors.

    Note we don't extend Preadditive C here, as Abelian C already extends it, and we'll need to have both typeclasses sometimes.
